# 校园饭卡管理系统 - 代码规范文档

## 1. 命名规范

### 1.1 包名命名规范
- 所有字母小写，多个单词直接用点（.）分隔
- 采用倒置的域名命名规则：com.公司名.项目名.模块名
- 示例：`com.example.campus_mealcardsystem.controller`

### 1.2 类名命名规范
- 使用大驼峰命名法（PascalCase）
- 类名应该是名词，避免使用动词
- 接口名以I开头，实现类名以Impl结尾（可选）
- 示例：`CardHolderController`、`CardHolderService`、`CardHolderServiceImpl`

### 1.3 方法名命名规范
- 使用小驼峰命名法（camelCase）
- 方法名应该是动词或动词短语
- 示例：`getById()`、`save()`、`updateById()`

### 1.4 变量名命名规范
- 使用小驼峰命名法（camelCase）
- 变量名应该有意义，避免使用单字母变量名（除循环变量外）
- 示例：`cardHolder`、`cardNumber`、`transactionType`

### 1.5 常量命名规范
- 所有字母大写，单词间用下划线分隔（UPPER_SNAKE_CASE）
- 示例：`MAX_RETRY_COUNT`、`DEFAULT_PAGE_SIZE`

### 1.6 数据库命名规范
- 表名使用小写字母和下划线分隔（snake_case）
- 字段名使用小写字母和下划线分隔（snake_case）
- 主键统一使用`id`
- 外键使用`表名_id`格式
- 示例：表名`card_holder`，字段名`card_number`

## 2. 注释规范

### 2.1 类注释规范
```
/**
 * 类功能描述
 * @author 作者
 * @date 创建日期
 * @version 版本号
 */
```

### 2.2 方法注释规范
```
/**
 * 方法功能描述
 * @param 参数名 参数描述
 * @return 返回值描述
 * @throws 异常类型 异常描述
 */
```

### 2.3 字段注释规范
```
/**
 * 字段描述
 */
```

### 2.4 代码块注释规范
- 使用单行注释`//`或多行注释`/* */`
- 注释应该说明"为什么这样做"，而不是"做了什么"
- 复杂逻辑应该添加注释说明

## 3. 代码格式规范

### 3.1 缩进规范
- 使用4个空格进行缩进
- 不使用Tab键

### 3.2 空格规范
- 运算符前后加空格：`a = b + c`
- 逗号后加空格：`method(a, b, c)`
- 关键字后加空格：`if (condition)`

### 3.3 换行规范
- 一行代码不超过120个字符
- 方法之间空一行
- 逻辑段落之间空一行

### 3.4 大括号规范
- 左大括号不换行，右大括号换行
- 示例：
```java
if (condition) {
    // code
} else {
    // code
}
```

## 4. 最佳实践

### 4.1 异常处理规范
- 捕获具体的异常，而不是捕获`Exception`
- 不要忽略异常，至少应该记录日志
- 使用自定义业务异常类

### 4.2 日志规范
- 使用SLF4J作为日志门面
- 合理使用日志级别：ERROR、WARN、INFO、DEBUG
- 日志应该包含足够的上下文信息

### 4.3 事务管理规范
- 使用`@Transactional`注解管理事务
- 指定合适的rollback条件
- 只读事务应该设置`readOnly = true`

### 4.4 安全规范
- 输入参数应该进行校验
- 敏感信息不应该明文存储
- 使用预编译SQL防止SQL注入

## 5. 分层规范

### 5.1 Controller层
- 负责接收请求和返回响应
- 不进行业务逻辑处理
- 使用`@RestController`注解
- 返回统一的结果格式（Result类）

### 5.2 Service层
- 负责业务逻辑处理
- 使用接口+实现类的模式
- 使用`@Service`注解
- 进行事务管理

### 5.3 Mapper层
- 负责数据库操作
- 继承MyBatis-Plus的BaseMapper
- 使用`@Mapper`注解或在启动类使用`@MapperScan`

### 5.4 Entity层
- 负责实体类定义
- 使用Lombok的`@Data`注解
- 使用MyBatis-Plus的注解进行表映射

## 6. 统一返回结果规范

### 6.1 Result类规范
- 使用泛型支持不同的返回数据类型
- 包含状态码（code）、提示信息（msg）、返回数据（data）
- 提供静态工厂方法创建成功和失败的Result对象

### 6.2 状态码规范
- 200：成功
- 400：参数错误
- 404：资源不存在
- 500：服务器内部错误
- 其他：业务错误码

## 7. 通用CRUD工具类规范

### 7.1 BaseEntity规范
- 包含通用字段：id、createTime、updateTime
- 使用MyBatis-Plus的注解

### 7.2 BaseMapper规范
- 继承MyBatis-Plus的BaseMapper
- 可以定义通用的查询方法

### 7.3 BaseService规范
- 定义通用的CRUD方法
- 使用泛型支持不同的实体类型

### 7.4 BaseServiceImpl规范
- 实现BaseService接口
- 继承MyBatis-Plus的ServiceImpl
- 使用`@Transactional`注解

## 8. 数据库配置规范

### 8.1 连接池配置
- 使用Druid连接池
- 配置初始连接数、最大连接数等参数
- 配置连接验证逻辑

### 8.2 MyBatis-Plus配置
- 配置Mapper扫描路径
- 配置实体类别名包路径
- 配置SQL日志输出

## 9. 代码提交规范

### 9.1 提交信息格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### 9.2 type类型
- feat：新功能
- fix：修复bug
- docs：文档修改
- style：代码格式修改
- refactor：代码重构
- test：测试用例修改
- chore：其他修改

### 9.3 示例
```
feat(card-holder): 添加持卡者注册功能

- 添加CardHolderController.register()方法
- 添加CardHolderService.save()方法

关联issue #123
```
